Overview

Multi-tenant dashboard for a voice-agent call business. An "agency" (root/admin) manages many client companies; each company has Retell voice agents whose call data is ingested (via n8n) into the app, billed through a threshold-based Stripe flow, and surfaced to per-company "staff" users.

Stack

  • Framework: Next.js 16.2.3, App Router, React 19.2.4 — RSC by default (pages are Server Components that gate by role, then delegate interactivity to co-located *-client.tsx Client Components).
  • Data: Drizzle ORM 0.45 over Postgres (pg); schema in lib/db/schema.ts, SQL migrations in drizzle/.
  • Auth: next-auth v5 beta, Credentials + JWT sessions (lib/auth.ts); page/route guard in proxy.ts (Next 16's renamed middleware).
  • UI: shadcn/ui on @base-ui/react, Tailwind v4, Geist fonts, lucide-react icons, sonner toasts.
  • Integrations: Stripe (billing), Retell (voice agents), n8n (ingestion + lookup), LeadSnap (lead routing).

Database Schema

Drizzle ORM over Postgres. 8 tables, 5 enums. All PKs are text UUIDs (crypto.randomUUID()). Schema: lib/db/schema.ts. Every relationship is one-to-many — there are no junction tables in the current model (the former company_agents M2M was dropped in migration 0011 in favour of retell_numbers, per ADR-010).

Session & protection

  • Session claims: { id, role, companyId, email } injected in the jwt → session callbacks (lib/auth.ts).
  • Page guard: proxy.ts matches all non-static requests. Public: /login, /audio-call, /api/auth, /api/webhooks, /api/external. Unauthed → /login. Page role gates redirect to /calls.
  • API guard: handled in-handler via requireRole(...) / getSessionUser() (lib/auth-helpers.ts) — proxy does not role-gate /api/*.
  • Core distinction: isAgencyRole(role) = root|admin (cross-company, companyId null) vs staff_admin|staff (scoped to one company).

Cron core logic — lib/billing/charge-cron.ts
  • recoverStuckCharging: frees companies stuck in charging > 1h with no live Stripe invoice.
  • Candidate selection: companies that are idle, have a payment method, and ≥ billingThresholdCalls (25) pending ledger entries; uses FOR UPDATE SKIP LOCKED.
  • chargeOneCompany: locks row, reserves pending ledger, creates local pending invoice, flips to charging, then creates Stripe invoice + item (idempotency keys); rolls back to idle/creation_failed on failure.
  • logSkippedCompanies: warns companies past threshold with no payment method (throttled 7 days); logs payment_pending/uncollectible skips.
